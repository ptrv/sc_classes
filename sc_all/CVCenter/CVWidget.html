CVWidget


inherits from: Object

dependencies: Conductor (Quarks)


A CVCenter is supposed to be a convenient GUI-class like an EZSlider and it should be possible to use it any GUI. However, its primary purpose is not the use as such. Much rather it should be understood as a module for CVCenter.


A CVWidget is GUI-representation of a CV (requires the Conductor-Quark to be installed). Instantiating a CVWidget as an element in a GUI automatically instantiates a CV that can be controlled by a slider (a Knob), a numerical input (a NumberBox) or a MIDI-device. CVWidgets have a built in MIDI-learn capability (see also CCResponder) - a slider or fader of a connected MIDI-device will automatically be connected if the MIDi-learn button is clicked in the widget.


Creation


*new(parent, cv, name, xy, widgetwidth, widgetheight, setup)

parent - the parent view: e.g. a Window or a TabbedView

cv - a CV (have a look at the CV helpfile for mor information on that)

name - each CVWidget will have a label displaying this name

xy - an instance of Point. The position of the CVWidget in its parent view

widgetwidth - by default this will be set to 52 pixels

widgetheight - by default this will be set to 137 pixels

setup - an array reflecting the setup of the connected external MIDI-device. If the setup consits of a single value (e.g.: 0) it will internally be converted into an array. 

The array mat contain the following slots: [ mode, mean ]

Currently two different modes are supported (the values represent mode :

0: MIDI-output is 0-127

1: MIDI-output is a mean-value +/- in-/decrement. If no value for mean is given in the setup-array 64 is assumed to be the default value for mean.

Default mode is 0.

Note: Setting a CVWidget to mode 1 will only make sense if your MIDI-device supports it. In this mode a MIDI-slider is assumed to output an in-/decremental value, not an absolute value between 0 and 127. However, the output is not assumed to be the increment/decrement only. Rather it will be the mean value +/- increment which makes it necessary to substract mean from the output (e.g. if the output equals 68 and mean is 64, a value of 64 is substracted and the result will be 4).



Example


(

var cv, win;

cv = CV.new(\boostcut.asSpec); 

win = Window("CVWidget");

~wdgt = CVWidget(win, cv, "CV test", 10@10);

win.front;

)


// change the Spec of the CV

// change is reflected in the widget immediately

// can also be done through the "edit Spec" button in the widget

~wdgt.spec_(\freq.asSpec)


// change the MIDI-setup

// change will only take effect after disconnecting and reconnecting the MIDI-device

~wdgt.midiSetUp_(1, 64)


about the GUI


The Elements within the GUI should be more or less selfexplanatory. However, one might ask why there the name is reflected twice - once as a non-editable label, once as an editable textfield. The reason for this is, that within CVCenter the label will reflect the key-name under which a CV has been added to CVCenter. The key may or may not reflect a key of a pattern or a control of a synth. However, a CV as part of CVCenter may be used in any context, so it may be useful to be able to add some addititional information, independantly from the key-name.

Within every CVWidget there is a blue button, labeled "L". This stands for "learn" - when clicked the button-color turns to red and the label becomes "X". In this state the first MIDI-slider that gets moved gets connected to the particular CVWidget resp. the parameters controlled by it.

Last but not least you will find a (non-functional) button named "MIDI" - this should become a shortcut to the widget's MIDI-setup. As it is currently not known to the author which different setups will be useful to provide this functionality shall be implemented at a later point.
